Duck's Services Support Handbook

Prepared from first-day training notes, team guidance, website/docs review, and ticket practice.

This handbook is a personal working reference for support use. It is meant to help with recognition, workflow, communication, and escalation. It does not replace team confirmation for unclear or advanced issues.

1. Role Overview

Duck's Services support is not about knowing every technical detail immediately. The core job is:

  • identify what type of ticket it is
  • understand what step failed
  • use the correct resource first
  • gather missing information
  • answer clearly when the issue is obvious
  • escalate when it is not obvious

Good support here means being calm, careful, and consistent. It does not mean guessing.

2. What Duck's Services Sells

Main categories:

  • `1PC products`
  • `DMA products`
  • `HWID Spoofers`
  • `Installation Services`

2.1 1PC Products

These are normal loader-based products that run on the customer's main gaming PC. Most beginner-friendly tickets fall into this category.

Typical customer flow:

buy -> key -> redeem -> download loader -> extract -> run as admin -> log in -> inject/load -> launch -> use menu

2.2 DMA Products

These are more advanced setups involving:

  • DMA card
  • firmware
  • second PC or mini PC
  • sometimes a fuser
  • sometimes extra accessories such as Makcu

DMA tickets require more caution. If a DMA answer is not obvious from the docs or official Discord commands, escalate early.

2.3 HWID Spoofers

Separate category. Some tickets may still be simple, such as download or access problems, but unclear technical issues should not be guessed on.

2.4 Installation Services

Some customers ask whether someone is available live to help them through AnyDesk or a similar remote support session. Do not promise live help until availability is confirmed.

3. Core Support Mindset

These are the most important habits:

  • clear and safe is better than fast and wrong
  • do not pretend to know an answer when uncertain
  • start with the official docs and support tool
  • use team confirmation for advanced or undocumented issues
  • the goal is not to look smart, but to be reliable

Strong support agents recognize patterns and know when to escalate.

4. The Basic 1PC Customer Journey

Most normal tickets can be understood by locating the failed step in the standard 1PC flow.

4.1 Buy

The customer purchases a product from the store.

Possible issues:

  • customer bought the wrong category
  • customer does not know what they purchased
  • customer asks what product is right for their game

4.2 Key / Redeem

The customer receives and redeems a license key.

Common issues:

  • wrong account
  • typo in the key
  • key already redeemed
  • redeemed in the wrong place
  • customer cannot find the key

4.3 Download Loader

The customer downloads the loader.

Common issues:

  • browser blocks file
  • antivirus blocks file
  • customer is on the wrong guide or page
  • broken or outdated link

4.4 Extract Files

The customer must extract the downloaded files.

Common issues:

  • running from a zip file
  • partial extraction
  • wrong location
  • folder naming/path problems

4.5 Run as Administrator

Many support cases involve permission or security issues.

Common issues:

  • not running as admin
  • Windows security blocks the file
  • downloaded file is marked as blocked

4.6 Log In / Launch / Load

The customer signs into the loader and attempts to launch or load it.

Common issues:

  • incorrect credentials
  • wrong account
  • key not redeemed correctly
  • product-specific loader issue
  • network/ISP-related issue

4.7 Use in Game

This is where injection/load or runtime issues may appear.

Common issues:

  • injection failed
  • product updating/down
  • antivirus interference
  • customer skipped a setup step

7. The Support Tool Workflow

This is a major part of the Duck's Services workflow.

For many normal setup issues:

  • tell the customer to use the support tool
  • ask them to send the reference/session code
  • search the code in Discord with `CTRL + F`
  • find the embed
  • review the system information

What the customer does:

  • runs the tool
  • fixes anything green so it turns red
  • presses `SEND TO SUPPORT`
  • sends the reference code

What support does:

  • looks up the code
  • reviews the embed
  • checks the obvious system settings first

8. Important Support Tool Checks

The team specifically said the biggest items to check are:

  • `Real-Time Protection`
  • `Vanguard Anti-Cheat`
  • `FaceIT Anti-Cheat`

Also commonly relevant:

  • `Firmware Virtualization`
  • `Secure Boot`
  • `UAC Status`

Based on the sample support tool embed, the desired safe state appears to be:

  • `Real-Time Protection` = Disabled
  • `Vanguard Anti-Cheat` = Not Detected
  • `FaceIT Anti-Cheat` = Not Detected
  • `Firmware Virtualization` = Disabled
  • `Secure Boot` = Disabled
  • `UAC` = Disabled

Important support tool note from the team:

If anything is green, fix it so it shows red

Meaning: if the tool shows a sensitive blocking/security item in the wrong state, that should be corrected before deeper troubleshooting.

9. Most Common 1PC Ticket Patterns

9.1 Key Not Working

Possible real causes:

  • wrong account
  • wrong redemption process
  • typo
  • already redeemed
  • wrong product/account pairing

Response style:

  • ask if key was redeemed
  • confirm correct account
  • ask for screenshot or exact error

9.2 Cannot Find or Download Loader

Possible real causes:

  • wrong page
  • wrong product guide
  • browser block
  • antivirus block
  • broken link

Response style:

  • ask what happens when they try
  • ask for screenshot
  • confirm product
  • ask if antivirus/browser blocked it

9.3 Loader Will Not Open

Possible real causes:

  • file blocked by Windows
  • antivirus quarantine
  • bad extraction
  • wrong folder
  • admin/security issue
  • loader-specific issue

9.4 Security Error / Please Open File Property

This usually points to Windows blocking the file after download.

Safe support direction:

  • right-click the file
  • open Properties
  • click `Unblock` if present
  • Apply / OK
  • run again as administrator
  • ensure files are extracted
  • ensure antivirus is disabled

9.5 Injection Failed

Possible real causes:

  • missed setup step
  • wrong launch order
  • AV/Defender interference
  • product updating/down
  • less obvious product-side issue

9.6 Error 521 / Site or Loader Access Error

This may be more like a site/service-side issue than a normal setup problem, especially if:

  • the site download itself throws the error
  • an older loader log shows the same error

In that case:

  • do not guess
  • avoid telling the customer to keep using the old loader
  • confirm with team

10. DMA Overview

DMA support is more advanced and should be approached carefully.

10.1 Common DMA Concepts

35t / 75t / 100t

DMA card tiers or models. Higher tiers generally support more advanced firmware and better performance.

Firmware

The logic/configuration layer used with the DMA card.

Second PC / Mini PC

A separate system used in the DMA setup.

Fuser

A device that combines output from two systems to one display for certain setups.

Makcu

An accessory/device in the DMA ecosystem. Follow official commands/docs and avoid improvising risky advice.

10.2 DMA Card and Firmware Command Shortcuts

DMA card commands:

  • `.dmainstall`
  • `.jtag`
  • `.ftdi`
  • `.speedtest`
  • `.dmatroubleshooting`

Firmware commands:

  • `.dnaid`
  • `.35tflash`
  • `.75tflash`
  • `.100tflash`

Makcu commands:

  • `.makcuflashing`
  • `.makcucables`
  • `.makcudriver`

Fuser commands:

  • `.fuser`
  • `.fusersetup`

The Discord command responses match the GitBook docs, so they can be used as fast references.

11. DMA Ticket Handling

For DMA tickets, follow this thinking process:

1. identify whether this is about the card, firmware, drivers, fuser, Makcu, or system setup

2. ask for the exact hardware model

3. ask for firmware name/version

4. ask for license key if relevant

5. ask for exact symptom or screenshot

6. use the matching Discord command or docs page

7. if still unclear, escalate

Do not freestyle advanced BIOS, firmware flashing, or hardware behavior unless the exact steps are clearly documented and you are confident they apply.

12. What Counts as Simple vs Advanced

12.1 Usually Okay to Handle First

  • key access questions
  • basic account/login confusion
  • simple download guidance
  • AV/Defender blocks
  • file property/unblock issue
  • simple setup order clarification
  • asking where to find the mini PC on the DMA page
  • asking how to use the official recommendation quiz

12.2 Escalate Earlier

  • DMA firmware uncertainty
  • driver issues after reinstalling Windows
  • BIOS setting advice when not fully sure
  • broken links
  • backend/server/site issues
  • unexplained crashes after correct setup
  • product-specific weird loader behavior
  • anything outside docs and official command outputs

13. Mini PC and Hardware Recommendation Notes

On the DMA page, the mini PC options listed are:

  • `Budget Mini PC`
  • `Premium Mini PC`

These are second-PC options, not DMA cards.

Important:

T100 / 100t is not the PC

It refers to the DMA card/setup tier.

When a customer asks which mini PC to use, first ask:

  • what games they plan to use it for
  • their budget
  • target resolution/refresh rate
  • whether they already have one of the PCs
  • whether they want a budget or smoother/higher-end setup

For a customer with:

  • ABI / EFT / Rust / CS2
  • 144Hz
  • mid-tier budget

The safer recommendation is usually the Premium Mini PC, because it gives more headroom and is a better match for a stronger DMA setup.

14. DMA Quiz / Recommendation System

The official .dmaquiz recommendation system is useful because:

  • it is consistent
  • it is official
  • it reduces the chance of making up the wrong recommendation

Best practice:

  • ask the customer to run `.dmaquiz`
  • use the returned recommendation
  • explain the result in simpler terms if needed

This is much safer than inventing a package recommendation from scratch.

15. Installation Service Questions

If a customer asks whether someone is available now for installation help through AnyDesk or similar, do not promise availability.

Safe response pattern:

  • identify that it is an installation service question
  • say you are checking availability
  • confirm with team

16. How to Read Any Ticket

Use this checklist:

1. what category is it?

2. what exact step failed?

3. what information is missing?

4. can the support tool or docs solve it first?

5. am I confident enough to answer?

6. if not, who should I escalate to?

This prevents panic and keeps replies accurate.

17. Best Questions to Ask

For normal 1PC tickets:

  • what product did you purchase?
  • what exactly happens when you try?
  • can you send a screenshot?
  • has the key already been redeemed?
  • are you on the correct account?
  • did you extract the files?
  • did antivirus or Defender block anything?
  • can you send the support tool reference code?

For DMA tickets:

  • what hardware model are you using?
  • what firmware are you using?
  • what is the exact issue?
  • when did it start?
  • did this start after a specific change?
  • can you send a screenshot or error?
  • what is your license key?

18. Reply Style

Replies should be:

  • polite
  • calm
  • short
  • direct
  • not overconfident
  • not overly technical unless necessary

18.1 Safe Reply Templates

General first reply

Hi, thanks for reaching out. Could you please send a screenshot or explain exactly what happens when you try that? Once I have that, I can check it more accurately for you.

Checking first

Hi, thanks for the details. I’m checking the correct steps for your setup now and will update you shortly.

Need more info

Please send your exact product, a screenshot of the error, and your reference code if available so I can check this properly.

Escalating

This appears to need a closer check, so I’m confirming it with the team first to make sure I give you the right guidance.

Broken link

I’m checking this for you now. Since you mentioned the link appears to be broken, I want to confirm the correct source first before having you try anything else.

19. Team Chat Etiquette

It is okay to ask teammates for help. The best way to do it is:

  • check the docs/context first
  • summarize the problem clearly
  • mention what you think it probably is
  • ask whether your next step is correct

Example:

Customer says the Arcane loader opens to a blank window and does not continue. Another loader runs as admin fine. I think this is more likely loader-specific, AV-related, extraction-related, or network-related rather than a BIOS issue. Could someone confirm the best next step before I advise them further?

That sounds much better than a vague help request.
